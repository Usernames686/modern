# CRMEB Modern 开发运维手册

## 技术栈

- 后端：Java 17、Spring Boot 3、MyBatis Plus、Maven。
- 后台：Vue 3、Vite、Element Plus、Pinia、Vue Router。
- H5：Vue 3、Vite、Vue Router、Axios。
- 基础服务：MySQL 8、Redis 7、Nginx。
- 部署：Docker Compose。

## 目录职责

- `backend/crmeb-modern-common`：通用返回、异常、工具和基础配置。
- `backend/crmeb-modern-service`：老库实体、Mapper、Service 和业务适配。
- `backend/crmeb-modern-admin-api`：后台 API，接口前缀保持 `/api/admin/**`。
- `backend/crmeb-modern-front-api`：H5 API，接口前缀保持 `/api/front/**`。
- `admin-web/src`：后台页面、菜单、路由、接口封装和组件。
- `app-web/src`：H5 页面、路由、接口封装和移动端样式。
- `legacy-deps`：运行依赖，不是源码开发入口。
- `legacy-source`：老项目源码快照，只用于对照和迁移。

## 环境变量

Docker 默认值在 `.env.example`：

- `CRMEB_DB_URL`：MySQL JDBC 地址。
- `CRMEB_DB_USERNAME` / `CRMEB_DB_PASSWORD`：数据库账号密码。
- `CRMEB_REDIS_HOST` / `CRMEB_REDIS_PORT` / `CRMEB_REDIS_DATABASE`：Redis 配置。
- `CRMEB_IMAGE_PATH`：上传资源目录，Docker 内为 `/data/upload/`。
- `CRMEB_ADMIN_SITE_URL`：后台站点地址。
- `CRMEB_APP_SITE_URL`：H5 站点地址。

前端本地代理：

- `CRMEB_ADMIN_API_TARGET`：后台开发服务代理目标，默认 `http://127.0.0.1:18080`。
- `CRMEB_FRONT_API_TARGET`：H5 开发服务代理目标，默认 `http://127.0.0.1:18081`。

## 数据库

当前继续使用老库结构：

- 数据库名：`single_open`
- 表前缀：`eb_`
- 初始化快照：`legacy-deps/db/single_open_current.sql`
- 原始 SQL：`legacy-deps/sql/single_open.sql`

不要在交付收口阶段大改表结构。如果必须加字段，先确认老系统兼容性，并同步更新 SQL 快照和文档。

## 上传资源

老系统图片路径继续兼容：

- `/crmebimage/**`
- `/public/**`

Docker 中由 Nginx 代理到 API，API 读取挂载目录 `/data/upload/`。本机直接启动 API 时，把 `CRMEB_IMAGE_PATH` 指向 `modern/legacy-deps/upload`。

## 常用命令

后端构建：

```bash
cd modern/backend
sh scripts/check-delivery.sh
```

后台构建：

```bash
cd modern/admin-web
npm run check:delivery
npm run build
```

H5 构建：

```bash
cd modern/app-web
npm run check:delivery
npm run build
```

全量交付审计：

```bash
cd modern
sh scripts/run-delivery-audit.sh full
```

Docker：

```bash
cd modern
docker compose up -d --build
docker compose ps
docker compose logs -f admin-api
docker compose logs -f front-api
docker compose down
```

## 功能迁移规则

1. 优先对照 `legacy-source/admin`、`legacy-source/app`、`legacy-source/crmeb`。
2. 可以参考 CRMEB 官方 GitHub 项目，但不要整仓覆盖。
3. 接口路径保持兼容：`/api/admin/**`、`/api/front/**`、`/api/public/**`。
4. 数据必须读写真实老表，不使用固定假数据冒充业务。
5. UI 不要求和老系统完全一致，但功能、字段、按钮和关键流程不能少。
6. 支付、退款、短信、微信发布、物流同步等高风险能力默认安全模式。

## Git 提交注意

不要提交：

- `node_modules/`
- `target/`
- `dist/`
- `unpackage/`
- `.local-logs/`
- `.env`
- 证书和私钥：`*.pem`、`*.key`、`*.p12`、`*.pfx`、`*.crt`、`*.cer`

需要提交：

- 源码改动。
- Docker 配置。
- 文档。
- 必要的 SQL 快照。
- 必要的上传图片和电商素材。

## 交付前检查清单

- `git status --short` 没有遗漏的源码和文档。
- GitHub 仓库能看到 `modern/` 完整代码、Docker、SQL 和素材。
- `docker compose up -d --build` 能启动六个服务。
- 后台 `admin / 123456` 能登录。
- H5 首页、分类、商品详情、购物车、订单、客服能打开。
- 后台新增商品后，H5 能看到。
- 图片不走旧机器绝对路径，不写死旧 IP。
- 真实支付、真实退款、短信、微信发布没有被误触发。
