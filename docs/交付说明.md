# CRMEB Modern 交付说明

本目录是 CRMEB Modern 的新版可交付工程，目标是用新技术栈提供完整后台、H5、API、数据库快照、上传资源和部署配置，同时兼容原系统的数据结构、接口路径和资源路径。

## 交付内容

- `backend/`：Java 17 + Spring Boot 3 多模块后端，包含后台 API 和 H5 API。
- `admin-web/`：Vue 3 + Vite + Element Plus 新后台。
- `app-web/`：Vue 3 + Vite 新 H5。
- `legacy-deps/`：新版运行需要的数据库快照、原始 SQL、上传图片和电商素材。
- `legacy-source/`：参考源码副本，用于后续对照和二次开发，不是新版运行入口。
- `docker-compose.yml`：一键编排 MySQL、Redis、后台 API、H5 API、后台页面、H5 页面。
- `docs/`：交付、部署、开发运维说明。

## 数据和素材

- 当前数据库快照：`legacy-deps/db/single_open_current.sql`
- 原始 SQL 副本：`legacy-deps/sql/`
- 上传资源和商品图片：`legacy-deps/upload/crmebimage/`

目前 `legacy-deps/upload` 约 161M，包含商品图、用户头像、富文本图片、首页素材和公共上传资源。`.pem`、`.key`、`.p12` 等证书类文件不会提交到 GitHub。

## Docker 启动

```bash
cd modern
cp .env.example .env
docker compose up -d --build
docker compose ps
```

首次启动时，MySQL 会用 `legacy-deps/db/single_open_current.sql` 初始化 `single_open` 数据库。API 容器会挂载 `legacy-deps/upload`，保证图片兼容路径 `/crmebimage/**` 能继续访问。

## 本地开发启动

后端：

```bash
cd modern/backend
./mvnw -pl crmeb-modern-admin-api -am -DskipTests package
./mvnw -pl crmeb-modern-front-api -am -DskipTests package
```

后台：

```bash
cd modern/admin-web
npm install
npm run dev
```

H5：

```bash
cd modern/app-web
npm install
npm run dev
```

本机直连后端时，建议设置上传目录：

```bash
export CRMEB_IMAGE_PATH="./legacy-deps/upload"
```

## 交付验收

推荐先跑统一脚本：

```bash
cd modern
sh scripts/run-delivery-audit.sh full
```

也可以分别执行：

```bash
cd modern/backend && sh scripts/check-delivery.sh
cd modern/admin-web && npm run check:delivery && npm run build
cd modern/app-web && npm run check:delivery && npm run build
```

重点人工验收：

- 后台登录、商品、订单、用户、营销、装修、财务、公众号、短信页面。
- H5 首页、分类、商品详情、购物车、订单、售后、客服、个人中心。
- 后台新增商品后，H5 能看到真实商品、图片、价格、库存和分类。
- H5 下单后，后台订单能看到。

## 安全边界

当前交付默认保守处理真实外部调用：

- 微信支付、支付宝支付、真实退款、短信发送、公众号发布、物流同步默认不直接触发。
- 页面入口和配置入口保留。
- 未配置生产参数时，接口应返回安全提示或 dry-run 结果。

上线前如需启用真实第三方能力，需要单独配置商户号、证书、回调域名、短信签名、公众号参数，并做专项验收。
