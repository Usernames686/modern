# 《CRMEB Modern》需求规格说明书

## 1. 文档目的

本文档描述 CRMEB Modern 项目的业务目标、功能范围、用户角色、运行要求和交付边界。文档结构参考原聚商盈一体化电商系统交付资料，但内容以当前 `modern` 新项目为准。

## 2. 项目背景

CRMEB Modern 是一体化电商系统的新版交付工程，包含后台管理端、H5 商城端、Java 后端、MySQL 数据库、上传素材和项目文档。项目目标是形成可单独交付的新目录：

```text
modern/
```

新版项目采用新技术栈实现电商业务能力，同时兼容原数据结构、接口路径和核心业务流程，降低交付和二次开发风险。

## 3. 总体目标

- 后台管理端使用 Vue 3 + Vite + Element Plus。
- H5 用户端使用 Vue 3 + Vite。
- 后端使用 Java 17 + Spring Boot 3。
- 数据库使用 `single_open` 和 `eb_*` 表。
- 接口路径使用 `/api/admin/**`、`/api/front/**`、`/api/public/**`。
- 上传资源使用 `/crmebimage/**` 兼容路径。
- 支持 Docker Compose 独立启动。
- 新项目目录内包含源码、数据库快照、运行素材、Docker 配置和交付文档。

## 4. 用户角色

### 4.1 平台管理员

管理商品、订单、用户、营销、装修、财务、分销、系统配置、应用配置和维护任务。

### 4.2 运营人员

维护首页、分类、活动、优惠券、文章、公众号菜单、关键字回复、短信配置和页面装修。

### 4.3 H5 用户

浏览商品、搜索商品、加入购物车、下单、支付、查看订单、申请售后、参与秒杀/拼团/砍价、维护个人资料。

### 4.4 系统维护人员

负责部署、数据库导入、日志排查、第三方配置、备份和交付验收。

## 5. 后台功能需求

### 5.1 控制台

- 展示销售额、订单量、用户数、商品数。
- 展示订单趋势、用户趋势、交易趋势。
- 提供核心模块快捷入口。

### 5.2 商品管理

- 商品列表、搜索、筛选、分页。
- 商品新增、编辑、上下架、回收站、恢复。
- 商品分类、规格模板、SKU、库存调整。
- 商品评价管理和商家回复。
- 运费模板、快递公司、城市数据。

### 5.3 订单管理

- 订单列表、订单详情、状态筛选。
- 订单备注、改价、发货、核销。
- 售后退款、拒绝退款、退款状态展示。
- 订单导出、打印、安全模式处理。

### 5.4 用户管理

- 用户列表、详情、筛选、分页。
- 用户分组、标签、等级。
- 积分、余额、经验、账单记录。

### 5.5 营销管理

- 优惠券列表、发放、领取记录。
- 秒杀活动和秒杀商品配置。
- 拼团商品配置、拼团列表。
- 砍价商品配置、砍价记录。
- 活动入口和活动订单桥接。

### 5.6 装修管理

- 首页轮播、菜单、商品 Tab、资讯配置。
- 个人中心菜单、个人中心轮播。
- 底部导航。
- 分类页配置。
- 微页面 DIY 数据保存和 H5 回显。

### 5.7 内容和应用

- 文章分类、文章列表、文章详情。
- 公众号菜单、关键字回复、模板消息。
- 短信模板、短信记录、OnePass 入口。
- 视频号入口保留，真实同步默认安全模式。

### 5.8 财务和分销

- 充值记录、提现申请、资金监控。
- 佣金记录、推广员列表、推广订单。
- 分销配置。

### 5.9 系统和维护

- 管理员、角色、权限菜单。
- 系统配置、主题色、附件管理。
- 定时任务、组合数据、表单配置、缓存清理。

## 6. H5 功能需求

### 6.1 首页

- 首页装修数据展示。
- 轮播图自动轮播和手动滑动。
- 快捷菜单、活动入口、推荐商品。
- 图片、文字、价格、跳转真实可用。

### 6.2 分类和商品

- 分类树展示。
- 分类商品按真实分类查询。
- 商品搜索、价格排序、销量排序。
- 商品详情展示轮播图、价格、库存、SKU、详情、评价。

### 6.3 购物车和订单

- 加入购物车、数量修改、删除、结算。
- 预下单、确认订单、创建订单。
- 订单列表、订单详情、物流、自提、确认收货。

### 6.4 支付和售后

- 余额支付和 0 元支付可用。
- 微信/支付宝入口保留，默认不触发真实支付。
- 申请退款、退款理由、售后列表、退款状态展示。

### 6.5 用户中心

- 用户资料、手机号、密码。
- 地址、收藏、优惠券、余额、积分、签到。
- 分销中心、推广订单、推广用户、佣金记录、提现。
- 客服对话页。

### 6.6 营销活动

- 秒杀列表和详情。
- 拼团列表、详情、团信息。
- 砍价列表、详情、发起和助力入口。

## 7. 非功能需求

- 独立性：`modern/` 应能作为独立交付目录使用。
- 兼容性：接口路径、数据库结构、图片路径保持稳定。
- 安全性：无 token 访问保护接口返回 401；真实第三方能力默认不误触发。
- 可维护性：文档、源码、数据、素材、Docker 配置集中在新项目目录。
- 可部署性：支持本地开发启动和 Docker Compose 部署。
- 可验收性：提供构建脚本、审计脚本和人工验收清单。

## 8. 交付边界

当前交付默认不启用真实微信支付、支付宝支付、真实退款、短信发送、公众号发布、物流同步。生产启用需要配置真实商户资料、证书、回调域名和专项验收。
