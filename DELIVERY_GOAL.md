# CRMEB Modern 功能级替代交付目标

## 目标

当前交付目标是用新技术栈完成老 CRMEB 的功能级替代：老业务能力、接口路径、数据结构和核心流程不能少，但 UI 不做像素级复刻，也不要求复刻老项目里不好用的交互；允许在 Vue3、Element Plus 和新版 H5 中重新整理成更清楚、更稳定、更适合交付演示的体验。

- 功能对齐：老项目能进入的页面、数据、接口、按钮和业务流程，交付版本不能消失；暂未重构完成的复杂模块必须旧版兼容保留。
- 体验优化：老 UI 里不好用、过时或混乱的地方可以在 modern 中重排、合并或美化，但必须保留关键入口、字段、状态、操作和可完成的业务动作。
- 快速交付：允许大量参考、复制、移植官方开源 CRMEB 和本地老项目代码，但不能整仓覆盖已跑通的 modern 工程。

## 参考源优先级

1. 本地老项目，作为最高优先级验收基准：
   - `admin/`：老后台路由、页面结构、表格字段、按钮、弹窗和业务流程。
   - `app/`：老用户端页面路径、组件行为、支付弹层、活动链路和字段结构。
   - `crmeb/`：老 Java 后端接口、Service、Mapper、任务队列、支付和第三方逻辑。
2. GitHub 官方 Java 版：`https://github.com/crmeb/crmeb_java`
   - 主参考源，用于 Java 后端、后台页面、用户端、支付、订单、营销活动和接口结构。
   - 与当前项目技术和目录结构最接近，可大量对照移植。
3. GitHub 官方 PHP 版：`https://github.com/crmeb/CRMEB`
   - 辅助参考源，用于页面 DIY、营销玩法完整性、用户端交互和 UI 完整性清单。
   - 不直接复制 PHP 后端逻辑到 Java，仅参考业务规则和界面行为。

## 复制粘贴规则

- 可以复制字段、接口参数、校验规则、表格列、弹窗结构、关键交互和业务流程。
- 可以复制老 Java 业务逻辑到 modern 后端，但必须适配 Java 17、Spring Boot 3、Jakarta、现有统一响应和鉴权。
- 可以复制老 Vue 页面结构到 Vue 3，但必须适配 Vite、Element Plus、当前请求封装和 modern 路由承接方式。
- 可以直接复用旧页面或旧接口，尤其是支付、页面装修、公众号、视频号、短信、物流同步等高风险模块。
- 不允许整仓覆盖 `modern/`，不允许为了复制开源代码破坏已跑通模块。
- 不允许用 mock 数据冒充迁移完成；所有列表和详情必须读取老库真实数据。
- 不允许在未明确验证前触发真实支付、真实退款、真实短信、真实微信发布或其他第三方写操作。

## 迁移工作流

1. 先找本地老页面、老接口、老表和官方 `crmeb_java` 对应实现。
2. 对照老页面列出字段、按钮、弹窗、接口、保存结构。
3. 判断处置方式：
   - 能快速补齐且风险低：复制移植到 modern。
   - 页面复杂但必须可用：旧版页面兼容保留。
   - 会触发资金、订单、库存或第三方：默认 dry-run 或老链路桥接。
4. 每次只做一个闭环：页面能进、数据能回显、操作能保存、刷新不丢、构建通过。
5. 完成后更新 `modern/MIGRATION.md`，说明参考源、迁移范围、未开放的高风险动作。

## 验收标准

- 用户能完成老 CRMEB 能完成的业务动作，关键字段和状态不丢。
- 老菜单入口不减少，父级菜单不会落到空白页。
- 已迁移页面没有占位页、白屏、无限加载或接口超时。
- 表格字段、筛选项、操作按钮、弹窗能力和移动端业务入口不比老项目少。
- 老数据、老图片、老上传路径和老接口路径继续可用。
- 核心链路能跑：登录、商品、用户、订单、营销活动配置、财务基础查询。
- 本地运行门禁必须覆盖浅层列表和深层详情：`runtime-smoke.mjs` 验证核心列表接口，`runtime-deep-smoke.mjs` 验证订单详情、商品详情、营销活动编辑详情、页面装修详情和 H5 商品详情/评价接口。
- 复杂功能即使没有 modern 完整重写，也能通过旧版兼容方式继续使用。
- 真实支付、真实退款、短信、微信发布、物流同步等动作必须有明确开关或继续走老项目稳定链路。

## 当前第一优先级

1. 全菜单巡检，持续消灭 `module-placeholder` 和明显“正在迁移中”入口。
2. 砍价商品编辑页按拼团/秒杀模式补齐多规格和 `type=2` 活动 SKU。
3. 订单桌面详情、发货、售后、退款按老页面补齐。
4. 用户端活动交易链路按老 `app/pages/activity` 逐个承接。
5. 支付桥接方案落地：modern 下单/充值入口可复用老支付链路，回调不在交付前重写。
